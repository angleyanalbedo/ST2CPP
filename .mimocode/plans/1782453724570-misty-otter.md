# ST2C VS Code Extension + Java LSP Server Plan

## Overview

为 ST2C 编译器创建完整的 VS Code IDE，基于 Java Language Server Protocol (LSP) 实现。

## 关键设计决策

**集成到现有 Maven 项目**：LSP Server 作为 `java/` 项目的新模块，而非独立项目。

理由：
1. **复用现有代码**：直接使用 ANTLR 语法、PLCVisitor、PLCTranslatorNew 等
2. **统一构建**：一个 `mvn package` 同时构建编译器和 LSP Server
3. **依赖管理**：LSP4J 依赖添加到现有 `pom.xml`
4. **维护简单**：一套代码，一处修改

## 功能列表

### Phase 1: 基础框架（1 周）

**1.1 Java LSP Server 框架**
- 使用 Eclipse LSP4J 库
- 实现 `LanguageServer` 接口
- 支持 JSON-RPC over stdin/stdout
- 启动参数：`java -jar st2c-lsp.jar`

**1.2 VS Code 扩展框架**
- 使用 `vscode-languageclient` 库
- 配置 LSP Client 连接 Java Server
- 注册语言：`st`
- 启动时自动启动 Java LSP Server

**1.3 语法高亮**
- TextMate grammar 文件
- 复用现有 ANTLR 语法规则

### Phase 2: 诊断功能（1 周）

**2.1 实时诊断**
- 文件修改时触发诊断
- 调用现有 Java 编译器
- 解析异常输出，提取错误信息
- 支持错误、警告、信息三种级别

**2.2 错误信息格式**
```json
{
  "range": {
    "start": {"line": 10, "character": 5},
    "end": {"line": 10, "character": 15}
  },
  "severity": 1,
  "message": "type mismatch : expected INT, got BOOL"
}
```

**2.3 性能优化**
- 文件修改后延迟 500ms 再触发诊断
- 缓存编译结果
- 增量分析（只重新分析修改的部分）

### Phase 3: 代码补全（1 周）

**3.1 关键字补全**
- PROGRAM, FUNCTION, FUNCTION_BLOCK, VAR, etc.
- IF, FOR, WHILE, REPEAT, CASE, etc.
- END_IF, END_FOR, END_WHILE, etc.

**3.2 类型补全**
- INT, REAL, BOOL, STRING, TIME
- ARRAY, STRUCT, ENUM

**3.3 函数补全**
- 内置函数：ABS, SQRT, SIN, COS, etc.
- 显示函数签名：`ABS(value : ANY_NUM) : ANY_NUM`

**3.4 功能块补全**
- TON, TOF, CTU, CTD, etc.
- 显示实例化语法

**3.5 代码片段**
- PROGRAM 模板
- FUNCTION 模板
- VAR 块模板
- 控制流模板（IF, FOR, WHILE）

### Phase 4: 悬停提示（1 周）

**4.1 关键字悬停**
- 显示关键字说明
- 示例：`PROGRAM` → "定义一个 PLC 程序块"

**4.2 函数悬停**
- 昽数函数签名和文档
- 示例：`ABS(x)` → "返回 x 的绝对值\n参数: x : ANY_NUM\n返回: ANY_NUM"

**4.3 变量悬停**
- 显示变量类型信息
- 需要符号表支持

### Phase 5: 跳转定义（1 周）

**5.1 变量跳转**
- 跳转到变量声明位置
- 支持跨文件跳转（VAR_EXTERNAL）

**5.2 函数跳转**
- 跳转到 FUNCTION 定义
- 跳转到 FUNCTION_BLOCK 定义

**5.3 类型跳转**
- 跳转到 TYPE 定义（STRUCT, ENUM）

### Phase 6: 高级功能（2 周）

**6.1 查找引用**
- 查找变量的所有引用
- 查找函数的所有调用

**6.2 文档符号**
- 大纲视图（Outline）
- 符号搜索（Ctrl+Shift+O）

**6.3 语义高亮**
- 基于类型信息的高亮
- 区分局部变量、全局变量、输入参数

**6.4 代码格式化**
- 自动缩进
- 关键字大写规范化

## 技术实现

```xml
    <!-- 现有依赖保持不变 -->

        <groupId>org.eclipse.lsp4j</groupId>
        <artifactId>org.eclipse.lsp4j</artifactId>
        <version>0.21.0</version>
    </dependency>
    
    <!-- 新增：LSP4J JSON -->
    <dependency>
        <groupId>org.eclipse.lsp4j</groupId>
        <artifactId>org.eclipse.lsp4j.jsonrpc</artifactId>

    </dependency>
```

```xml
        </archive>
    </configuration>
</plugin>
```

### Step 2: 实现 Java LSP Server

**2.1 `ST2CLanguageServer.java`**
```java
public class ST2CLanguageServer implements LanguageServer {
    private TextDocumentService textDocumentService;
    private WorkspaceService workspaceService;
    
    @Override
    public CompletableFuture<InitializeResult> initialize(InitializeParams params) {
        // 返回服务器能力
        ServerCapabilities capabilities = new ServerCapabilities();
        capabilities.setTextDocumentSync(TextDocumentSyncKind.Full);
        capabilities.setCompletionProvider(new CompletionOptions());
        capabilities.setHoverProvider(true);
        capabilities.setDefinitionProvider(true);
        return CompletableFuture.completedFuture(new InitializeResult(capabilities));
    }
}
```

**2.2 `ST2CTextDocumentService.java`**
- 实现 `didOpen`, `didChange`, `didSave` 处理文档事件
- 调用现有编译器进行分析
- 返回诊断信息

**2.3 `DiagnosticAnalyzer.java`**
- 调用现有 `PLCVisitor` 进行语义分析
- 捕获 `PLCException` 异常
- 转换为 LSP Diagnostic 格式

**2.4 `CompletionAnalyzer.java`**
- 基于 ANTLR 语法提供关键字补全
- 基于符号表提供变量、函数补全

**3.2 `extension.ts`**
```typescript
import * as path from 'path';
import { LanguageClient, TransportKind } from 'vscode-languageclient/node';

let client: LanguageClient;

export function activate(context: vscode.ExtensionContext) {
    const serverModule = context.asAbsolutePath(
        path.join('..', 'java', 'target', 'st2c-jar-with-dependencies.jar')
    );
    
    const serverOptions = {
        run: { command: 'java', args: ['-jar', serverModule] },
        debug: { command: 'java', args: ['-jar', serverModule] }
    };
    
    const clientOptions = {
        documentSelector: [{ scheme: 'file', language: 'st' }]
    };
    
    client = new LanguageClient('st2c', serverOptions, clientOptions);
    client.start();
}
```

## 注意事项

1. **性能**: Java LSP Server 启动较慢（500ms-1s），但后续响应快（100-300ms）
2. **内存**: Java 进程占用 50-100MB，可接受
3. **兼容性**: 需要 JDK 8+ 运行环境
4. **分发**: 可以打包为独立 JAR，无需用户安装 Java 开发环境
5. **构建**: `mvn package` 同时生成编译器和 LSP Server 的 JAR
